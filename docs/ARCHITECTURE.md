## 1. Обзор проекта

**Уголки** (Corners / Halma) — браузерная настольная игра «Игрок vs Компьютер» (или «Игрок vs Игрок»). Цель — первым переместить все свои фишки из своего угла доски в противоположный.

### Стек технологий

| Технология | Версия | Назначение |
|---|---|---|
| React | 19 | UI-фреймворк |
| Tailwind CSS | 4 | Стилизация через utility-классы |
| Vite | 7 | Сборка и dev-сервер |
| Yandex Games SDK | — | Интеграция с платформой Яндекс Игр (реклама, лидерборды, сохранение данных) |

### Структура файлов

```
src/
├── main.jsx                  # Точка входа, инициализация SDK
├── App.jsx                   # Корневой компонент (роутинг lobby/game)
```

## 2. Правила игры

### Доска и стартовые позиции

Игра ведётся на квадратной доске размером от **8×8** до **16×16**. В двух противоположных углах располагаются квадратные зоны с фишками:

| Пресет | Размер доски | Размер зоны (K) | Фишек у каждого |
|---|---|---|---|
| small | 8×8 | 3×3 | 9 |
| medium | 12×12 | 4×4 | 16 |
| large | 16×16 | 5×5 | 25 |

- **Игрок 1 (ИИ)** занимает верхний левый угол (ячейки `[0..K-1, 0..K-1]`).
- **Игрок 2 (Человек)** занимает нижний правый угол (ячейки `[rows-K..rows-1, cols-K..cols-1]`).

### Движение фишек

Фишки перемещаются по **4 направлениям** (вверх, вниз, влево, вправо — без диагоналей):

1. **Шаг** — перемещение на одну свободную соседнюю клетку.
2. **Прыжок** — перепрыгивание через соседнюю занятую фишку (любого игрока) на свободную клетку за ней. Прыжки можно объединять в **цепочку** за один ход.

Поиск цепочки прыжков реализован через **BFS** (поиск в ширину).

### Условие победы

Побеждает тот, кто первым заполнит **все** клетки целевой зоны (противоположного угла) своими фишками. Целевая зона Игрока 1 — зона старта Игрока 2, и наоборот.

### Первый ход

Определяется настройкой `firstMove`:
- `'computer'` — первым ходит ИИ
- `'random'` — выбирается случайно (по умолчанию)

## 3. Архитектура приложения

### Дерево компонентов

```
App
├── Lobby                     # screen.type === 'lobby'
│   └── BoardPreview
└── Game                      # screen.type === 'game'
    ├── Board
    │   ├── Cell (×rows×cols)
    │   └── GhostPiece
    ├── InfoPanel
    └── GameOver              # Показывается при state.winner !== null
```

### Потоки данных

1. **Навигация**: `App` использует `useReducer` для переключения между `Lobby` и `Game`. При нажатии «Начать игру» `Lobby` вызывает `onStart(settings)`, `App` переключает экран на `Game` с переданными настройками.

2. **Игровой стейт**: `Game` создаёт `useReducer(gameReducer, settings, createInitialGameState)`. Весь стейт игры хранится в одном объекте и мутируется через dispatch actions.

3. **Ход игрока**: клик по клетке → `Board.handleClick` → `dispatch(SELECT_PIECE)` или `onPlayerMove` → анимация через `useAnimation` → `dispatch(MAKE_MOVE)`.

4. **Ход ИИ**: `useAiTurn` слушает изменение `state.currentPlayer`. Когда ходит AI_PLAYER (1), запускается `getBestMove` через `setTimeout` → результат анимируется → `dispatch(MAKE_MOVE_AI)`.

5. **Статистика**: `usePlayerStats` загружает/сохраняет данные через Yandex SDK (`getPlayer().getData()`/`setData()`). При завершении игры `Game` вызывает `onGameEnd(state)`.

## 4. Игровая логика — `src/game/logic.js`

### `getCornerK(cornerSize): number`

Возвращает размер стороны стартовой зоны по имени пресета:
- `'small'` → 3, `'medium'` → 4, `'large'` → 5

### `getMaxCornerSize(rows, cols): string`

Определяет максимально допустимый `cornerSize` для заданного размера доски. Ограничение: количество фишек не должно превышать 30% площади доски.

### `createBoard(rows, cols, cornerSize): { board, zones }`

Создаёт начальное состояние доски:
- `board` — двумерный массив `rows × cols`, заполненный `null` (пустая клетка), `1` (фишка Игрока 1) или `2` (фишка Игрока 2).
- `zones` — объект с двумя `Set`:
  - `player1` — строки ключей `"r,c"` клеток стартовой зоны Игрока 1 (верхний левый угол).
  - `player2` — клетки стартовой зоны Игрока 2 (нижний правый угол).

### `getValidMoves(board, row, col, zones, currentPlayer): { steps, jumps, jumpPaths }`

Вычисляет все допустимые ходы для фишки на позиции `(row, col)`:

- **steps** — массив `[row, col]` клеток, куда можно шагнуть (4 направления, свободная клетка).
- **jumps** — массив `[row, col]` клеток, куда можно допрыгнуть (через BFS по цепочке прыжков).
- **jumpPaths** — `Map<string, Array<[row, col]>>` — полные пути прыжков для каждой целевой клетки. Ключ — `"row,col"`, значение — массив промежуточных и конечных точек.

**Алгоритм BFS для прыжков:**
1. Начальная позиция добавляется в `visited`.
2. Очередь содержит `[текущийRow, текущийCol, путь]`.
3. Для каждого направления проверяется: если на `(cr+dr, cc+dc)` стоит любая фишка, а `(cr+2*dr, cc+2*dc)` свободна и не посещена — это допустимый прыжок.
4. Новая позиция добавляется в очередь с обновлённым путём.

### `makeMove(board, fromRow, fromCol, toRow, toCol): board`

Создаёт копию доски и перемещает фишку из `(fromRow, fromCol)` в `(toRow, toCol)`. Исходная клетка обнуляется.

### `checkWin(board, zones): 1 | 2 | null`

Проверяет условие победы:
- Игрок 1 побеждает, если **все** клетки `zones.player2` заняты фишками `1`.
- Игрок 2 побеждает, если **все** клетки `zones.player1` заняты фишками `2`.
- Возвращает номер победителя или `null`.

### `getJumpPath(jumpPaths, targetRow, targetCol): Array`

Извлекает путь прыжка из `jumpPaths` Map по целевой позиции. Используется для визуализации пути при наведении.

## 5. AI алгоритм — `src/game/ai.js`

### Общая архитектура

ИИ работает на трёх уровнях сложности:
- **easy** — жадный алгоритм: перебирает все ходы, оценивает каждый, выбирает лучший с добавлением случайного шума (±5).
- **medium** — Minimax с альфа-бета отсечением, глубина 2–3.
- **hard** — Minimax с альфа-бета отсечением, глубина 3–4.

Глубина адаптируется к размеру доски: на досках ≥12 глубина уменьшается на 1.

### Дедлайн по времени

Все уровни ограничены дедлайном `AI_DEADLINE_MS = 2000 мс`. Если время истекло, minimax прерывает обход и возвращает текущую лучшую оценку.

### `manhattanToTarget(r, c, rows, cols, player): number`

Манхэттенское расстояние от позиции `(r, c)` до целевого угла:
- Для Игрока 1: `(rows-1-r) + (cols-1-c)` (до нижнего правого угла).
- Для Игрока 2: `r + c` (до верхнего левого угла).

### `getAllMoves(board, zones, player): Array<Move>`

Генерирует все допустимые ходы для игрока. Перебирает все фишки, для каждой вызывает `getValidMoves` и собирает шаги и прыжки.

### Оценочная функция `evaluate(board, zones, aiPlayer, difficulty, aiMoveCount)`

Возвращает числовую оценку позиции для ИИ. Чем выше — тем лучше для ИИ. Компоненты:

| Компонент | Формула | Описание |
|---|---|---|
| **Суммарное расстояние** | `−aiDist × 3 × urgency` | Сумма Манхэттенских расстояний всех фишек ИИ до цели. `urgency` растёт после 20-го хода (+3% за ход). |
| **Отстающая фишка** | `−maxPieceDist × 8` | Штраф за самую далёкую от цели фишку — стимулирует двигать отстающих. |
| **Фишки в цели** | `+aiInFinish × 30` | Бонус за каждую фишку, уже стоящую в целевой зоне. |
| **Победа/поражение** | `±100 000` | Если все фишки в цели — бонус `WIN`; если противник закончил — штраф. |
| **Кластеризация** | `+clusterScore` | Бонус за близость продвинутых фишек (расстояние ≤ 2 в Чебышёве). Только для фишек, прошедших >40% пути. |
| **Разброс (дисперсия)** | `−excess² × 0.5` | Штраф за фишки, сильно отстающие от среднего расстояния. |
| **Фишки на старте** | `−20` за фишку | Штраф за фишки, всё ещё стоящие в своей стартовой зоне. |

**Дополнительные компоненты по сложности:**

| Сложность | Доп. компоненты |
|---|---|
| **medium** | `+oppDist × 0.5`, `−oppInFinish × 10` |
| **hard** | `+oppDist × 1.5`, `−oppInFinish × 20`, бонус +2 за блокировку фишек противника (расстояние Чебышёва ≤ 2) |

### `sortMoves(moves, board, zones, player): Array<Move>`

Сортирует ходы для улучшения альфа-бета отсечения. Приоритет:
1. Ходы в целевую зону (финиш) — выше.
2. Ходы с большим прогрессом (увеличение `row+col` для Игрока 1, уменьшение для Игрока 2).
3. Более длинные ходы (по Манхэттену) — выше.

### `hashBoard(board): string`

Сериализует доску в строку для отслеживания повторов позиций. Формат: строки разделены `|`, клетки: `0` (пустая), `1`, `2`.

### `minimax(board, zones, depth, alpha, beta, maximizing, aiPlayer, difficulty, deadline, aiMoveCount)`

Классический Minimax с альфа-бета отсечением:
- **Базовые случаи**: дедлайн, победа/поражение, `depth === 0`.
- **Maximizing** (ход ИИ): ищет ход с максимальной оценкой, обновляет `alpha`.
- **Minimizing** (ход противника): ищет ход с минимальной оценкой, обновляет `beta`.
- Отсечение: `beta ≤ alpha` → прерывание ветки.
- Таймаут: если `Date.now() > deadline` → возврат текущей оценки.

### Эскалирующий штраф за повтор позиции `getRepeatPenalty(hash, positionFrequency)`

| Повторов | Штраф |
|---|---|
| 0 | 0 |
| 1 | −500 |
| 2 | −5 000 |
| 3+ | −100 000 |

Частота позиций подсчитывается по `hashBoard` из истории ходов.

### `getBestMove(board, zones, aiPlayer, difficulty, positionFrequency, aiMoveCount): Move | null`

Главная точка входа. Алгоритм:

1. Определяет глубину поиска по сложности и размеру доски.
2. Для `easy`: жадная оценка каждого хода + случайный шум ± 5 + штраф за повтор.
3. Для `medium`/`hard`: запускает minimax по всем отсортированным ходам верхнего уровня, добавляет штраф за повтор к результату.
4. Прерывается по дедлайну.
5. Возвращает лучший ход или `null` если ходов нет.

## 6. Reducer — `src/game/reducer.js`

### Начальное состояние `createInitialGameState(settings)`

```js
{
  board,                    // Двумерный массив доски
  zones,                    // { player1: Set, player2: Set }
  currentPlayer,            // 1 или 2 (определяется firstMove)
  selectedPiece: null,      // { row, col } выбранной фишки
  validMoves: null,         // Результат getValidMoves для выбранной фишки
  moveCount: { 1: 0, 2: 0 }, // Счётчик ходов каждого игрока
  winner: null,             // null, 1 или 2
  settings,                 // Настройки из лобби
  history: [],              // Массив снимков для отмены хода
  startTime: Date.now(),    // Начало партии (для таймера)
  freeUndosLeft: 2,         // Оставшиеся бесплатные отмены
}
```

### Actions

#### `SELECT_PIECE { row, col }`
Выбирает фишку текущего игрока. Если на `(row, col)` стоит не его фишка — сбрасывает выбор. Вычисляет `validMoves` для выбранной фишки.

#### `MAKE_MOVE { row, col }`
Ход человека. Перемещает `selectedPiece` на `(row, col)`. Переключает `currentPlayer`, инкрементирует `moveCount`, проверяет `checkWin`, сохраняет снимок в `history`.

#### `MAKE_MOVE_AI { fromRow, fromCol, toRow, toCol }`
Ход ИИ. Аналогичен `MAKE_MOVE`, но принимает координаты «откуда-куда» напрямую (у ИИ нет `selectedPiece`).

#### `UNDO`
Отмена хода. Против компьютера откатывает **2 хода** (ход ИИ + ход игрока), иначе — 1 ход. Уменьшает `freeUndosLeft`. Восстанавливает `board`, `currentPlayer`, `moveCount` из `history`.

#### `SURRENDER`
Сдача. Текущий игрок проигрывает, победитель — его оппонент.

#### `DESELECT`
Снимает выбор фишки (`selectedPiece = null`, `validMoves = null`).

#### `RESET`
Пересоздаёт начальное состояние с теми же настройками.

## 8. Утилиты — `src/utils.js`

### `formatTime(ms): string`

Форматирует миллисекунды в строку `MM:SS` с ведущими нулями.

### `pieceGradient(player): string`

Возвращает CSS `radial-gradient` для объёмной фишки с бликом:
- Внутренний блик: белый ellipse в верхней левой трети.
- Основной градиент: от `--color-p1`/`--color-p2` до `--color-p1-dark`/`--color-p2-dark`.

### `buildMovePath(selectedPiece, targetRow, targetCol, validMoves): Array<[row, col]>`

Собирает путь анимации фишки:
- Если это прыжок (ключ есть в `jumpPaths`) — возвращает `[стартовая, ...промежуточные, конечная]`.
- Если шаг — возвращает `[стартовая, конечная]`.

- `useAiTurn()` — автоматический ход ИИ.

Кнопки управления:
- **Отменить** — вызывает `dispatch(UNDO)`. Если бесплатные отмены кончились — показывает рекламу через `showRewarded()`.
- **Сдаться** — `dispatch(SURRENDER)`.
- **В меню** — показывает interstitial-рекламу, затем возвращает в Lobby.

Блокировка ввода: ходы игрока заблокированы, когда ходит ИИ (`isPlayerTurnBlocked`) или идёт анимация (`animating.current`).

При появлении `state.winner` показывается `GameOver` модал.

### `Board` — `src/components/Board.jsx`

Отрисовка доски через CSS Grid. Пропсы: `state`, `dispatch`, `isLocked`, `hiddenCell`, `onPlayerMove`, `ghostPiece`.

**Логика взаимодействия:**
- `handleClick(row, col)`:
  - Если выбрана фишка и клик по допустимому ходу → вызов `onPlayerMove` с путём.
  - Если клик по своей фишке → `dispatch(SELECT_PIECE)`.
  - Иначе → `dispatch(DESELECT)`.
- `handleMouseEnter(row, col)`: при наведении на клетку с допустимым прыжком показывает путь прыжка (подсвечивает промежуточные клетки).

Мемоизация: `stepSet` и `jumpSet` кэшируются через `useMemo` для быстрой проверки `has()`.

### `Cell` — `src/components/Cell.jsx`

Одна клетка доски. Определяет визуальное состояние через комбинацию пропсов:
- `isCheckerDark` — чередование цвета клеток (шахматный паттерн).
- `isP1Zone` / `isP2Zone` — подсветка стартовых зон.
- `isStep` / `isJump` — подсветка допустимых ходов (окрашена в цвет текущего игрока).
- `isOnPath` — подсветка пути при наведении на прыжок.
- `isSelected` — выделение выбранной фишки (кольцо accent).
- `isHidden` — скрытие фишки во время анимации (ghost piece занимает её место).
- `isPlayerPiece` — hover-эффект (scale) только для фишек текущего игрока.

Фишки отрисовываются как `div` с `radial-gradient` (через `pieceGradient`), `border` и `box-shadow` для объёмности. Точки-подсказки (30% круг) показывают допустимые ходы.

### `GhostPiece` — `src/components/GhostPiece.jsx`

Абсолютно позиционированный элемент поверх доски. Позиционируется через процентные `left`/`top` на основе `row`, `col`, `rows`, `cols`. CSS `transition` на `left`/`top` (200ms ease-in-out) обеспечивает плавное перемещение.

### `InfoPanel` — `src/components/InfoPanel.jsx`

Боковая (на десктопе) / горизонтальная (на мобильных) панель с информацией:
- Текущий ход (с цветной точкой).
- Индикатор «думает...» (три анимированных точки bounce) для хода ИИ.
- Счётчик ходов обоих игроков.
- Таймер партии (MM:SS).
- Прогресс-бары заполнения целевой зоны для каждого игрока.
- Информация о размере поля и сложности.

Адаптивность: на мобильных (`lg:hidden`) показываются сокращённые лейблы, на десктопе — полные.

Модальное окно при завершении партии:

## 10. Хуки

### `useAnimation` — `src/hooks/useAnimation.js`

Управляет анимацией перемещения фишек.

**Возвращает:**
- `ghostPiece` — `{ player, row, col } | null` — текущая позиция анимируемой фишки.
- `hiddenCell` — `{ row, col } | null` — клетка, фишка на которой скрыта (чтобы не было дубля).
- `animate(path, player, onDone)` — запускает анимацию по пути.
- `animating` — `useRef(false)` — флаг активной анимации.

**Алгоритм:**
1. Скрывает фишку на стартовой клетке (`hiddenCell`).
2. Создаёт `ghostPiece` на стартовой позиции.
3. По таймеру (`ANIMATION_STEP_MS = 250ms`) перемещает `ghostPiece` на следующую точку пути.
4. CSS `transition` на `GhostPiece` анимирует перемещение.
5. После последнего шага убирает `ghostPiece` и `hiddenCell`, вызывает `onDone`.

Используется `requestAnimationFrame` × 2 перед первым шагом, чтобы браузер успел отрисовать начальную позицию до старта transition.

### `useGameTimer` — `src/hooks/useGameTimer.js`

Таймер партии.

- Обновляет `elapsed` каждую секунду через `setInterval`.
- Паузится при `isPaused === true` (когда есть победитель).
- `resetTimer()` — сброс к 0.

### `useAiTurn` — `src/hooks/useAiTurn.js`

Автоматическое управление ходом ИИ.

**Возвращает:**
- `isThinking` — флаг «ИИ думает».
- `resetAi()` — сброс состояния.

**Логика (useEffect):**
1. Срабатывает при изменении `state.currentPlayer`, `state.winner`, `state.board`.
2. Проверяет: есть ли ИИ, нет ли победителя, сейчас ход ИИ, не запущен ли уже расчёт.
3. Через `setTimeout(AI_MOVE_DELAY = 400ms)` — задержка для UX:
   a. Подсчитывает `positionFrequency` из `state.history`.
   b. Вызывает `getBestMove(...)`.
   c. Строит путь анимации через `buildMovePath`.
   d. Запускает `animate(path, AI_PLAYER, callback)`.
   e. В callback: `dispatch(MAKE_MOVE_AI)`, сброс `isThinking`.
4. Cleanup: очищает таймер при размонтировании.

### `usePlayerStats` — `src/hooks/usePlayerStats.js`

Статистика игрока с персистентностью через Yandex SDK.

**Возвращает:**
- `playerStats` — объект `DEFAULT_STATS` с данными из SDK.
- `recordGameResult(state)` — обновляет статистику после завершения игры.

**`recordGameResult`:**
- Определяет, победил ли игрок (`state.winner !== AI_PLAYER`).
- Инкрементирует `totalGames`, `wins` или `losses`.
- При победе обновляет `bestMoves` и `bestTime` (если текущий результат лучше).
- При победе отправляет `submitScore('main', moves)` в лидерборд.
- Сохраняет через `savePlayerData(newStats)`.

### Yandex Games SDK (`src/sdk/yandex.js`)

Обёртка над Yandex Games SDK с автоматическим fallback на dev-стабы:
- Если `window.YaGames` доступен — инициализирует реальный SDK.
- Иначе — создаёт `createDevStub()` с `console.log`-заглушками.

**API:**
- `initYandexSDK()` — инициализация, вызывается в `main.jsx`.
- `gameReady()` — сигнал платформе о готовности игры (`LoadingAPI.ready()`).
- `gameplayStart()` / `gameplayStop()` — управление GameplayAPI (платформа знает, когда можно показывать рекламу).
- `showInterstitial()` — полноэкранная реклама (Promise). Останавливает gameplay, возобновляет после закрытия.
- `showRewarded()` — реклама за награду (Promise → boolean). Используется для дополнительных отмен ходов.
- `loadPlayerData()` / `savePlayerData(data)` — чтение/запись данных игрока через `getPlayer().getData()`/`setData()`.
- `submitScore(name, score)` — отправка результата в лидерборд.
- `getLeaderboardEntries(name, opts)` — получение записей лидерборда.
